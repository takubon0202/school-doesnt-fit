# 学校に合わない子のための本紹介Webページプロジェクト

## 概要
不登校・発達特性のある子どもとその保護者向けの本を紹介するWebページを開発するプロジェクトです。

## ターゲットペルソナ
- 不登校経験のある小中高生
- 発達特性の偏りがある子ども
- 不登校や発達障害の子を持つ保護者

## デザインポリシー
- AIっぽいデザインは避ける（絵文字の乱用、紫色基調のデザインなど）
- オリジナリティと合理性のバランス
- 温かみがありつつも信頼感のあるデザイン
- ターゲットに寄り添う優しい雰囲気

## サブエージェントシステム
このプロジェクトでは以下のサブエージェントを使用して開発を進めます：

### 1. デザイナー (`/design`)
ページデザインの設計と指示を担当

### 2. コーダー (`/coding`)
HTML/CSS/JavaScriptの実装を担当

### 3. 画像生成ディレクター (`/image`)
画像生成の指示書を作成

### 4. バックエンド (`/backend`)
必要に応じてバックエンド機能を実装

### 5. デバッガー (`/debug`)
コードの品質チェックと修正

### 6. ペルソナ評価者 (`/persona`)
ターゲットペルソナ視点での評価

### 7. SEO・事業評価者 (`/evaluate`)
SEOと事業観点での評価

### 8. パイプライン実行 (`/pipeline`)
全サブエージェントを順番に実行

### 9. レスポンシブ専門 (`/responsive`)
iOS/Android/iPadOS対応の徹底検証と実装を担当

## 開発フロー
1. `/design` - デザイン設計
2. `/coding` - 実装
3. `/image` - 画像指示書作成
4. `/debug` - デバッグ
5. `/persona` - ペルソナ評価
6. `/evaluate` - SEO・事業評価
7. 評価結果に基づき改善サイクル

## AI CLI連携
このプロジェクトでは複数のAI CLIを統合して開発を行います：

### Codex CLI (`/codex`)
OpenAI Codex CLIを使用したコード生成・エラー解決・デザイン修正
- **サブスクリプション**: ChatGPT Plus ($20/月)
- **モデル**: gpt-5.3-codex
- **用途**: コーディング、CSS修正、デザイン調整、リサーチ

### Gemini CLI (`/gemini`)
Google Gemini CLIを使用したコード生成・Webリサーチ・デザイン修正
- **サブスクリプション**: Gemini AI Pro ($19.99/月)
- **モデル**: gemini-3-pro-preview / gemini-3-flash-preview
- **用途**: Webリサーチ（Google Search内蔵）、コーディング、デザイン分析

### 使用量上限ルール
使用量上限に達した場合：
1. 即座に報告
2. 該当CLIの使用を停止
3. 代替CLIを提案

## Agent Teams（実験的機能）

### 概要
複数のClaude Codeインスタンスがチームとして連携して動作します。
`CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS=1` で有効化済み。

### チーム構成（3名体制）

```
┌─────────────────────────────────────────────────┐
│              Agent Team Structure                │
├─────────────────────────────────────────────────┤
│                                                 │
│          ┌──────────────────────┐               │
│          │  Lead: 統括リーダー    │               │
│          │  (調整・品質管理)      │               │
│          └──────────┬───────────┘               │
│                     │                           │
│          ┌──────────┴───────────┐               │
│          │                      │               │
│  ┌───────┴────────┐  ┌─────────┴──────┐        │
│  │ Member 1:       │  │ Member 2:       │       │
│  │ デザインエンジニア │  │ QAエンジニア     │       │
│  │ (UI/UX・CSS)    │  │ (デバッグ・修正)  │       │
│  └────────────────┘  └────────────────┘        │
│                                                 │
└─────────────────────────────────────────────────┘
```

### Member 1: デザインエンジニア
**役割**: UI/UXデザインとCSS実装の専門家
**担当領域**:
- CSSスタイリング・レイアウト改善
- glassmorphism・Anti-Gravityデザインの調整
- アニメーション・トランジションの実装
- レスポンシブデザインの最適化
- カラーパレット・タイポグラフィの管理
- デザインスペックの作成・更新

**所有ファイル**:
- `src/css/style.css` のデザイン関連セクション
- `src/css/anti_gravity_utilities.css`
- `src/css/new_styles.css`
- `design-spec-*.md`

### Member 2: QAエンジニア
**役割**: コード品質・エラー修正・パフォーマンスの専門家
**担当領域**:
- HTML/CSS/JavaScriptのバグ修正
- アクセシビリティ検証・改善
- パフォーマンス最適化
- クロスブラウザ互換性テスト
- SEO技術要件の確認
- モバイルデバイス対応の検証

**所有ファイル**:
- `src/js/main.js`
- `src/css/mobile_fixes.css`
- `debug-report*.md`
- `index.html` の構造的修正

### Lead: 統括リーダー
**役割**: チーム全体の調整と品質管理
**担当領域**:
- タスクの分割と割り当て
- Member 1とMember 2の成果物レビュー
- 全体の整合性チェック
- 最終的な品質判定（ペルソナ評価・SEO評価の基準達成確認）
- コンフリクト解消
- プロジェクト全体のスケジュール管理

**権限**:
- デリゲートモード使用（自らコードを書かず調整に集中）
- 両メンバーの成果物を統合
- 改善サイクルの管理（最大3回）

### チーム起動方法

チームを起動するには、以下のようにリクエストします：

```
3人のエージェントチームを作成してください：
1. デザインエンジニア - UI/UXとCSS実装を担当。Sonnetモデルを使用。
2. QAエンジニア - バグ修正とパフォーマンス最適化を担当。Sonnetモデルを使用。
リーダーは全体の調整と品質管理に集中してください（デリゲートモード）。
```

### チーム運用ルール

1. **ファイル競合の回避**: 各メンバーは自分の所有ファイルのみ編集する
2. **プラン承認**: 大きな変更は実装前にリーダーの承認を得る
3. **品質ゲート**: 両評価（ペルソナ+SEO）で70点以上を達成すること
4. **コミュニケーション**: メンバー間でメッセージを送り合い、変更の影響を共有する
5. **完了報告**: タスク完了時は必ずリーダーに報告する
